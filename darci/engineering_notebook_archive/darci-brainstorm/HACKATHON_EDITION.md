## 🚀 HACKATHON Workflows

### Workflow 1: Idea → MVP (Hours 0-4)
```
1. DARCI brainstorm features
   → "Generate 10 feature ideas for [theme]"

2. Auto-create task list
   → "Create MVP tasks, prioritize by impact"

3. Scaffold project
   → "Init repo, add dependencies, create structure"

4. Generate boilerplate
   → "Create [feature] with [tech stack]"

5. Deploy preview
   → "Deploy to Vercel, share link"
```

### Workflow 2: Feature Sprint (Hours 4-24)
```
1. Pick next high-impact feature
   → "What's the next P0 task?"

2. Generate code
   → "Implement [feature] using [library]"

3. Quick test
   → "Does it work? (yes/no)"

4. Commit & deploy
   → "Commit, push, auto-deploy"

5. Update team
   → "Slack: Feature X shipped!"
```

### Workflow 3: Crisis Mode (Anytime)
```
🔴 PANIC BUTTON ACTIVATED

1. Stop all non-critical tasks
2. Identify the fire
3. Auto-generate fix
4. Test fix (minimal)
5. Deploy emergency patch
6. Post-mortem (later)
```

### Workflow 4: Demo Prep (Hours 36-48)
```
1. Generate demo script
   → "Create 3-min demo flow"

2. Create pitch deck
   → "10 slides: problem, solution, market"

3. Record demo video
   → "Screen record + voiceover"

4. Polish UI
   → "Fix visible bugs only"

5. Generate metrics
   → "Impressive dashboard (fake ok)"

6. Rehearse
   → "Run through demo 3x"
```
